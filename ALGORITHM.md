# 미디어 팩트체크는 어떻게 작동할까?
> 학생을 위한 VAPM v3.0 알고리즘 안내서 (5대 검증 행동 기반)

이 문서는 우리 웹앱이 **여러분의 체크리스트**와 **AI(Gemini)** 그리고 **베이지안(Bayesian) 학습**을 어떻게 한 데 묶어 미디어 자료를 평가하는지 풀어 설명합니다. 수학을 잘 몰라도 괜찮아요. 모든 수식 옆에는 "이게 무슨 뜻인지"를 친절하게 적어두었습니다.

---

## 1. 한 줄 요약

> "내 체크리스트로 평가 → 선생님 평가와 비교 → 매번 조금씩 내 검증 행동 가중치를 다듬는다."

이 과정을 반복하면 처음에는 똑같이 중요하다고 봤던 5대 검증 행동의 가중치가, 점점 **여러분만의 균형 잡힌 기준**으로 자리 잡습니다. 그리고 5개 행동 각각의 **마스터리(숙달도)** 가 함께 추적돼요.

---

## 2. 왜 5대 검증 행동인가요?

이전 버전(IPFM v2.0)은 **IFCN 5대 강령**을 기준으로 했어요. 그런데 IFCN 강령은 본래 **팩트체커 조직이 지켜야 할 윤리 강령**이지, 학생이 미디어 자료를 평가할 때 직접 수행하는 행동이 아니에요.

VAPM v3.0은 **너희가 미디어 자료를 만났을 때 실제로 해야 하는 5가지 검증 행동**으로 재설계했어요. 각 항목이 "이렇게 행동해보자"라는 구체적 가이드예요.

| 코드 | 검증 행동 | 한마디로 |
|---|---|---|
| **V1** | 출처 확인 (Source Check) | 이 사이트 진짜야? |
| **V2** | 저자 확인 (Author Check) | 누가 썼고 사람 맞아? |
| **V3** | 콘텐츠 교차 확인 (Content Cross-check) | 다른 데서도 확인돼? |
| **V4** | 이미지·영상 확인 (Visual Verification) | 이 사진·영상 진짜야? |
| **V5** | 감정 반응 점검 (Emotional Reaction Check) | 나 너무 흥분한 거 아니야? (메타인지) |

> **V6(사용자 정의)**: 5대 검증 행동 어디에도 안 들어맞는 질문은 자동으로 V6으로 분류돼요. AI가 헷갈려서 임의로 끼워 맞추지 않게 하기 위함입니다. V6 항목은 가중치 학습엔 직접 영향을 안 주지만 여러분의 체크리스트엔 그대로 남아 있습니다.

> **V5는 왜 다른 4개랑 다를까?**
> V1~V4는 **자료를 보는** 행동이고, V5는 **자기 자신을 보는** 행동이에요. 강한 감정에 휩싸이면 우리 뇌는 비판적 사고를 잘 못 해요. 그래서 가짜뉴스를 만드는 사람들은 일부러 분노·공포를 자극해서 우리가 생각 없이 공유하게 만들어요. 이걸 인식하는 것 자체가 가장 강력한 방어막이에요.

## 3. 전체 흐름 한눈에 보기

```
[1] 내 체크리스트 만들기  ─────────────────►  질문마다 1~5점 루브릭 정의
                                                    │
                                                    ▼
[2] AI가 질문을 5대 검증 행동에 자동 매핑  ────►  "이 질문은 V3(콘텐츠 교차)이네"
                                                    │
                                                    ▼
[3] 교사가 등록한 미디어를 내가 평가
        + 같은 미디어를 교사도 평가
                  │
                  ▼ (학생 vs 교사 점수 비교 = "격차 gap")
[4] 베이지안 가중치 갱신
        : 격차가 큰 행동의 가중치를 살짝 조정
                  │
                  ▼ 여러 번 반복 → 내 가중치 W가 안정됨
                  │
                  ▼
[5] 새 미디어를 Gemini가 5대 검증 행동으로 1~5점 평가
        (V4는 본문에 시각 자료 언급이 없으면 N/A로 자동 제외)
                  │
                  ▼
[6] 최종 점수 = (W × Gemini 점수)를 50점 환산
        + 95% 신뢰구간(±오차범위)
                  │
                  ▼
[7] 결과 화면: 그대로 "수용" or 직접 점수 고쳐 "정교화"
                  │
                  ▼
[8] 검증 행동별 마스터리 갱신
        🌟 = 잘하는 행동, ⚠️ = 약점 행동
```

## 4. 7층 구조로 다시 풀면

```
Layer 7 │ 최종 점수(50점 만점) + 95% 신뢰구간 + 검증 행동 마스터리   ← 결과 카드
Layer 6 │ 베이지안 가중치 갱신 (μ 평균, σ 불확실성) + Mastery 산출    ← 학습이 일어나는 곳
Layer 5 │ 교사와의 격차 분석 → 메타인지 피드백 카드
Layer 4 │ 학생 가중치를 점수에 적용 (V4 N/A인 경우 4개로 재정규화)
Layer 3 │ Gemini가 5대 검증 행동에 1~5점 부여 (단일 호출)
Layer 2 │ 체크리스트 항목 → 5대 검증 행동 자동 매핑(캐싱)
Layer 1 │ 학생이 직접 만든 커스텀 체크리스트
```

> 위로 갈수록 "결과", 아래로 갈수록 "학생 본인의 입력"에 가깝습니다. 이 구조가 우리 모델의 이름 **VAPM (Verification Action-based Progressive fact-check Model)** 의 의미예요.
> - **Verification Action-based** = 5대 검증 행동에 정렬
> - **Progressive** = 평가를 반복할수록 점진적으로 내 가중치와 마스터리가 학습됨

## 5. 핵심 개념 5가지

### (1) 가중치 W = { V1: {μ, σ}, …, V5: {μ, σ} }

각 검증 행동마다 **두 개의 숫자**를 들고 다닙니다.

- **μ (뮤, mu)** = "이 행동을 얼마나 중요하게 볼 것인가"의 평균값. 5개의 μ를 다 더하면 1.
- **σ (시그마, sigma)** = "이 가중치에 대해 얼마나 자신 있는가"의 불확실성. 클수록 불확실, 작을수록 확신.

처음에는 모두에게 똑같이 1/5씩(=0.20) 나눠줍니다. (균등 가중치, σ=0.15)

### (2) 격차 (gap)

같은 미디어에 대해 **교사 점수 − 내 점수**.
- gap > 0 → 내가 너무 박하게 봤다
- gap < 0 → 내가 너무 후하게 봤다

### (3) 학습률 η(t)

> `η(t) = max(0.05, 0.2 × exp(−0.05 × t))`

- t = 누적 학습 데이터 수
- 처음(t=0)엔 0.2로 크게 움직이고, 데이터가 쌓일수록 천천히 작아집니다 (최소 0.05).
- 직관: **처음엔 빠르게 배우고, 나중엔 신중하게 미세 조정.**

### (4) 베이지안 갱신 식

> `μ_i_new = μ_i_old + η × gap_i × score_i / Σ(score_j²)`

말로 풀면:
1. 교사와 격차가 컸던 행동일수록 → 더 많이 움직인다
2. 그 미디어에서 강하게 두드러진 행동일수록 → 더 의미 있는 신호로 본다
3. 전체 점수 크기로 나눠서 → 모든 행동을 공평하게 비교한다

그리고 매 갱신마다 σ는 0.95배로 줄여요. **데이터가 쌓일수록 내 판단 기준이 안정화된다**는 의미입니다.

### (5) 마스터리 (Mastery)

> `Mastery(V_i) = (1 − σ_i) × (1 − |avg_gap_i| / 4)`

각 검증 행동을 얼마나 안정적으로 수행하는지 0~1 사이 값으로 산출.

말로 풀면:
- σ가 작아질수록 (= 자신감이 생길수록) 첫 항이 1에 가까워짐
- 평균 격차가 작아질수록 (= 교사와 점수 차이가 적을수록) 둘째 항이 1에 가까워짐
- 두 조건이 함께 충족돼야 마스터리가 높아짐

대시보드에서 🌟(80% 이상)/⚠️(40% 미만) 색상으로 잘하는 행동과 약점 행동이 한눈에 보여요.

## 6. 학습 단계 정책

| 단계 | 누적 데이터 t | 무슨 일이 일어나나 |
|---|---|---|
| 🌱 Cold Start | t < 3 | 균등 가중치만 사용. 베이즈 갱신 비활성. (자료가 너무 적으면 섣불리 학습하지 않음) |
| 🌿 워밍업 | 3 ≤ t < 5 | 신중하게 갱신 시작 |
| 🌳 Bayesian 활성 | t ≥ 5 | σ 본격 감소, 가중치 정교화 가속 |

> 왜 단계를 나눴을까? **표본이 너무 적을 때 학습하면 한 번의 우연한 격차에 휘둘려버리기** 때문이에요. 통계학에서 "작은 표본의 위험"이라고 부르는 문제를 막기 위함입니다.

## 7. 최종 점수와 신뢰구간

Gemini가 5대 검증 행동에 1~5점을 매기면:

> `S = ( Σ μ_i × score_i / Σ μ_active ) × 10`        → 0~50점 만점

여기서 `Σ μ_active`는 실제로 점수가 매겨진 행동들의 가중치 합이에요. **V4(이미지·영상 확인)가 N/A**인 경우, 자동으로 나머지 4개 행동으로 정규화됩니다.

> 💡 **균등 가중치 사례 — 같은 식이라는 것 확인하기**
> 5개 행동 모두 μ=0.2, 점수가 각 (3, 4, 5, 4, 3)이면
> · 가중평균식: (0.2×3 + 0.2×4 + 0.2×5 + 0.2×4 + 0.2×3) / 1.0 × 10 = 3.8 × 10 = **38점**
> · 합산식: (3+4+5+4+3) ÷ 25 × 50 = 19 × 2 = **38점**
> 두 결과가 정확히 일치해요.
>
> **V4 N/A 사례** — V1~V3, V5만 있고 점수 (3, 4, 5, 3), μ는 모두 0.2면:
> · 가중평균식: (0.2×3 + 0.2×4 + 0.2×5 + 0.2×3) / 0.8 × 10 = 3.0/0.8 × 10 = 3.75 × 10 = **37.5점**
> · "4개 합산 × 2.5"식: (3+4+5+3) × 2.5 = 15 × 2.5 = **37.5점**
> 활성 가중치만으로 자동 재정규화돼서 두 식이 또 일치해요.

여기에 σ를 이용해 **얼마나 흔들릴 수 있는지**를 계산합니다.

> `Var(S) = 100 × Σ score_i² × σ_i²`     (활성 행동만)
> `95% 신뢰구간 = S ± 1.96 × √Var(S)`

> **왜 ±인가?** 우리 모델은 단정적으로 "이 자료는 38점!"이라고 외치지 않아요. "38점인데, 95% 확률로 35.4 ~ 40.6 사이일 거야"처럼 **불확실성을 함께 보여주는 것**이 통계적으로 더 정직한 표현이기 때문입니다. 1.96은 정규분포에서 95% 구간을 만들 때 쓰는 표준값이에요.

### 점수 구간별 해석 (50점 만점 기준)

| 점수 구간 | 해석 |
|---|---|
| 45~50점 | 매우 신뢰할 만한 자료. 학습·인용 가능 수준. |
| 35~44점 | 대체로 신뢰할 만하나 일부 영역 보완 필요. |
| 25~34점 | 비판적 검토 후 부분적 활용 가능. 추가 확인 필수. |
| 15~24점 | 신뢰성 의심. 다른 자료 우선 참고 권장. |
| 0~14점 | 부정확하거나 편향된 자료. 인용·공유 부적절. |

## 8. "수용" vs "정교화"

결과를 본 학생은 두 가지 선택을 합니다.

| 선택 | 학습 데이터에 추가되는 값 | 가중치에 미치는 영향 |
|---|---|---|
| 🟢 **수용** | Gemini 점수 그대로 저장 | σ만 약간 ↓ (관찰이 누적된 효과) |
| 🟡 **정교화** | 학생이 직접 수정한 점수 | `bayesianUpdate` 발동, **학습률 η × 1.5 가속**, 학생-Gemini 격차를 신호로 사용 |

> "AI 결과에 동의하지 않는다"는 행위 자체가 **여러분의 비판적 판단**이고, 그것이 모델을 더 정교하게 다듬는 신호가 됩니다.

## 9. 메타인지 피드백 카드

여러 번의 격차 패턴을 분석해서 자동으로 만드는 진단 카드입니다 (`generateFeedbackCards`).

| 카드 | 트리거 조건 | 의미 |
|---|---|---|
| **Over** | 평균 gap < −0.5 | 교사보다 평균 0.5점 이상 **후하게** 평가하는 경향 |
| **Under** | 평균 gap > +0.5 | 교사보다 평균 0.5점 이상 **박하게** 평가하는 경향 |
| **Inconsistent** | gap 분산 > 1.0 | 평가 결과가 **흔들리는** 경향 (기준이 일관되지 않음) |

각 카드에는 검증 행동 · 진단 · 자기 점검 가이드 · 구체적 개선 행동이 표시돼요.

> **메타인지(metacognition)** 란 "내가 어떻게 생각하고 있는지를 생각하는 능력"을 말합니다. 이 카드는 여러분 자신의 평가 습관을 거울처럼 비춰주는 도구예요.

## 10. 수렴도 (Convergence Score)

> `C = 1 − || W_student − W_teacher_implicit || / √5`

- 0 ~ 1 사이 값.
- 1에 가까울수록 **여러분의 가중치 분포가 교사가 (암묵적으로) 중요하게 본 행동 분포와 비슷해졌다**는 뜻.
- "성장 그래프"처럼 학생 대시보드에 표시됩니다.
- (분모의 √5는 5개 검증 행동이라서 그래요. 이전 7차원 모델에서는 √7이었습니다.)

## 11. V4(이미지·영상 확인) N/A 자동 처리

본문에 사진·영상·그래프·차트·스크린샷 언급이 전혀 없는 텍스트 자료는 V4를 평가할 단서가 없어요. 그래서:

1. Gemini 프록시가 본문을 살펴보고 시각 자료 단서가 없으면 V4를 `score: null, skipped: true`로 반환합니다.
2. `computeFinalScore`는 V4를 빼고 활성 가중치(V1, V2, V3, V5)만으로 자동 재정규화합니다.
3. 결과 화면 V4 카드는 "해당 없음 (N/A)"으로 표시되고 가중치는 회색으로 보여줍니다.
4. 베이지안 갱신과 마스터리 산출에서도 V4는 자동으로 제외됩니다.

본문에 시각 자료 언급이 조금이라도 있으면(예: "위 그래프", "사진에 보이는") V4도 정상 평가돼요.

## 12. 미디어 유형별 권장 가중치 프리셋

자료 유형에 따라 어떤 검증 행동이 더 중요한지 다릅니다. 코드에 5종 프리셋이 내장돼 있어요 (`MEDIA_TYPE_PRESETS` in `src/utils/hpfm.js`):

| 미디어 유형 | V1 | V2 | V3 | V4 | V5 | 강조점 |
|------------|-----|-----|-----|-----|-----|---------|
| 뉴스 기사 | 0.20 | 0.15 | **0.30** | 0.15 | 0.20 | 교차 검증 우선 |
| SNS 게시물 | 0.15 | **0.30** | 0.15 | 0.20 | 0.20 | 봇·익명 점검 우선 |
| 영상 콘텐츠 | 0.15 | 0.20 | 0.20 | **0.30** | 0.15 | 딥페이크·이미지 우선 |
| 광고/홍보 | 0.20 | 0.15 | 0.20 | 0.15 | **0.30** | 감정 자극 우선 |
| 정부 공식 발표 | **0.30** | 0.25 | 0.25 | 0.10 | 0.10 | 출처 정확성 우선 |

## 13. 한 번의 평가, 어떻게 굴러갈까? (예시 시나리오)

**상황**: 학생이 "출처 확인·콘텐츠 교차 확인"을 가장 중요하게 본다고 가정. 누적 평가 5회.

1. 학생이 미디어 A를 자기 체크리스트로 평가 → 검증 행동별 점수 집계 → `V1=4, V3=3, V4=N/A(이미지 없음), ...`
2. 교사도 미디어 A를 평가 (5대 검증 행동 기반 5개 디폴트 항목) → `V1=2, V3=4, ...`
3. **gap = 교사 − 학생** → `V1=−2 (학생이 너무 후함), V3=+1 (학생이 약간 박함)`
4. 베이지안 갱신:
   - V1의 μ가 살짝 ↓ (출처를 너무 후하게 봤으니 그 영향력 보정)
   - V3의 μ가 살짝 ↑ (콘텐츠 교차는 박했으니 그 영향력 강화)
5. 매 행동 σ ×0.95 → 전반적인 자신감 ↑
6. 정규화 (Σμ = 1)
7. 마스터리 산출: `Mastery(V_i) = (1-σ_i) × (1-|avg_gap_i|/4)`
8. 누적된 격차들을 보고 "Over V1" 같은 피드백 카드 자동 생성
9. 새 미디어 B를 Gemini로 평가 → 갱신된 W로 50점 환산 → `34.2 ± 4.1점`
10. 학생이 결과를 보고 "수용" 또는 "정교화" 선택 → 다시 7번으로 순환

## 14. 자주 묻는 질문

**Q. AI(Gemini)가 틀린 점수를 주면 어떻게 되나요?**
A. 그래서 "정교화" 버튼이 있습니다. 여러분이 동의하지 않으면 점수를 직접 수정하세요. 그 차이가 모델 학습 신호로 들어가서 다음 번엔 더 여러분에게 맞는 가중치가 됩니다.

**Q. 처음 한두 번은 점수가 이상해요.**
A. 정상입니다. 누적 데이터가 5개 미만이면 모델이 의도적으로 "균등 가중치(0.20)"를 유지해요(Cold Start). 평가가 5번 이상 쌓이면 본격적으로 여러분 기준이 반영됩니다.

**Q. 5대 검증 행동 중에 못 들어가는 질문이 있어요.**
A. AI가 자동으로 V6(사용자 정의)로 분류합니다. V6 항목은 가중치 학습엔 직접 영향을 안 주지만 여러분의 체크리스트엔 그대로 남아 있어요.

**Q. 신뢰구간이 너무 넓어요.**
A. 평가 횟수가 적어서 σ가 큰 상태입니다. 모델링을 반복할수록 σ가 줄어들고 신뢰구간도 좁아져요.

**Q. V4(이미지·영상)가 항상 "해당 없음"이에요.**
A. 본문에 사진·영상·그래프 언급이 없는 텍스트 자료라서 그래요. 본문에 "위 그래프", "사진에 보이는" 같은 시각 자료 인용이 있으면 V4도 평가됩니다. 영상 콘텐츠나 SNS 이미지 게시물을 평가하려면 본문에 그 시각 정보를 함께 적어 넣으면 더 정확해져요.

**Q. 마스터리가 뭐예요?**
A. 각 검증 행동을 얼마나 안정적으로 수행하는지를 0~1로 보여주는 점수예요. σ(자신감)와 격차(정확도)가 함께 좋아질 때 올라갑니다. 🌟(80% 이상)는 잘하는 행동, ⚠️(40% 미만)는 더 연습이 필요한 약점 행동이에요.

**Q. 왜 IFCN 강령에서 5대 검증 행동으로 바뀌었어요?**
A. IFCN 강령은 본래 **팩트체커 조직이 지켜야 할 윤리 기준**이지, 학생이 미디어 자료를 평가할 때 직접 수행하는 행동이 아니에요. VAPM v3.0은 "이렇게 행동해보자"라는 구체적 가이드(출처/저자/콘텐츠/이미지/감정)로 재설계해서 미디어 리터러시 교육에 더 적합하게 만든 거예요. 이전 7차원·5차원 데이터는 자동 변환되어 옮겨집니다.

## 15. 더 알아보기 (출처·참고문헌)

이 알고리즘은 **5대 검증 행동(SIFT 모델 계열)** 과 **베이지안 추론**을 결합했습니다.

### 미디어 리터러시 프레임워크 (메인 표준)
- **SIFT (The Four Moves)**: Mike Caulfield, *SIFT (The Four Moves)*, 2019. Stop / Investigate / Find / Trace.
  공식 글: <https://hapgood.us/2019/06/19/sift-the-four-moves/>
  → V5(Stop), V1·V2(Investigate), V3(Find), V4(Trace) 행동에 영감을 줌.
- **한국언론진흥재단 미디어 리터러시 교육 자료**: 디지털 환경에서의 검증 행동 가이드.
- **EU vs Disinformation / EUFactCheck**: 유럽위원회 산하 디지털 시민 역량 가이드. 봇·딥페이크·역이미지 검색 등 디지털 환경 위협에 대한 점검 행동을 권장.

### 보조 프레임워크 (체크리스트 설계 시 참고)
- **CRAAP Test**: Sarah Blakeslee, *The CRAAP Test*, LOEX Quarterly 31(3), 2004. 정보 평가의 5요소(Currency, Relevance, Authority, Accuracy, Purpose).
- **IFCN Code of Principles**: International Fact-Checking Network (Poynter Institute). 본 모델 v2.0의 토대였으며, V1·V2(출처·저자 투명성)는 IFCN 강령 1·3과 정신을 공유합니다.

### 통계·기계학습
- **Bayes' Theorem**: Thomas Bayes, *An Essay towards solving a Problem in the Doctrine of Chances*, Philosophical Transactions of the Royal Society of London, 1763 (사후 발표). 새 증거가 들어왔을 때 신념(가중치)을 어떻게 업데이트할지에 대한 수학적 토대.
- **Online / Stochastic Gradient Update**: Robbins & Monro, *A Stochastic Approximation Method*, Annals of Mathematical Statistics, 1951. 우리 모델의 학습률 스케줄(η decay)이 따르는 고전 결과.
- **95% Confidence Interval**: 정규분포 가정 하의 신뢰구간 계산. 표준 통계학 교과서 (예: Wasserman, *All of Statistics*, Springer 2004, §6).

## 16. 학습 활동 아이디어 (선생님과 함께)

1. **같은 기사, 다른 체크리스트**: 모둠별로 다른 검증 행동을 강조하는 체크리스트를 만든 뒤, 같은 기사에 대해 어떤 결과 차이가 나오는지 비교.
2. **가중치 변화 추적**: 모델링을 5회, 10회, 20회 했을 때 각 검증 행동(V1~V5)의 μ 변화 그래프를 직접 그려보기.
3. **AI vs 인간**: Gemini의 5대 검증 행동 평가를 그대로 수용한 결과와, 정교화로 수정한 결과를 비교해 "AI를 어디까지 신뢰할 것인가" 토론.
4. **메타인지 일지**: 피드백 카드(Over/Under/Inconsistent)를 받으면, 그 원인을 자기 언어로 정리하고 다음 평가에서 어떻게 다르게 해볼지 미리 적기.
5. **자료 유형별 강조점 실습**: 뉴스 기사(V1·V3), SNS 게시물(V2·V5), 영상(V4), 광고(V5), 정부 발표(V1·V3) 별로 어느 행동이 더 중요한지 토론하고 직접 가중치 조정. `MEDIA_TYPE_PRESETS`로 비교 가능.
6. **검증 행동 시연**: 교사가 학생들 앞에서 도메인 검색(V1) → 작성자 구글링(V2) → 다른 매체 찾기(V3) → 역이미지 검색(V4) → 자기 감정 점검(V5)을 실제로 시연.
7. **마스터리 진단**: 학기 중 마스터리 그래프를 보고 가장 약한 검증 행동을 집중 학습. 학기 말에 다시 비교.
