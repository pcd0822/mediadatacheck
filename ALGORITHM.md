# 미디어 팩트체크는 어떻게 작동할까?
> 학생을 위한 HPFM 알고리즘 안내서

이 문서는 우리 웹앱이 **여러분의 체크리스트**와 **AI(Gemini)** 그리고 **베이지안(Bayesian) 학습**을 어떻게 한 데 묶어 미디어 자료를 평가하는지 풀어 설명합니다. 수학을 잘 몰라도 괜찮아요. 모든 수식 옆에는 "이게 무슨 뜻인지"를 친절하게 적어두었습니다.

---

## 1. 한 줄 요약

> "내 체크리스트로 평가 → 선생님 평가와 비교 → 매번 조금씩 내 평가 기준을 다듬는다."

이 과정을 반복하면 처음에는 모두 똑같이 중요하다고 봤던 7가지 평가 차원의 가중치가, 점점 **여러분만의 균형 잡힌 기준**으로 자리 잡습니다.

---

## 2. 왜 7개 차원으로 나누나요?

전 세계 미디어 리터러시 교육에서 자주 쓰이는 **공인된 4가지 프레임워크**를 합쳐 7개의 핵심 질문으로 정리했습니다.

| 코드 | 차원 | 이런 걸 봐요 | 출처 프레임워크 |
|---|---|---|---|
| **D1** | 출처 권위성 (Authority) | 누가 썼지? 어느 매체지? 믿을 만한가? | CRAAP, SIFT |
| **D2** | 내용 정확성 (Accuracy) | 사실로 검증 가능한가? 통계 출처가 명시됐나? | CRAAP, FEVER |
| **D3** | 시의성 (Currency) | 너무 오래된 자료는 아닌가? | CRAAP |
| **D4** | 근거 제시 (Evidence) | 인용·1차 자료·데이터가 있나? | FEVER, SIFT |
| **D5** | 편향성·목적 (Bias) | 한쪽 입장만? 광고성? 정파적 어휘? | CRAAP, IFCN |
| **D6** | 언어 건전성 (Language) | 자극적·과장된 표현? 클릭베이트? | IFCN |
| **D7** | 검증 가능성 (Verifiability) | 다른 출처로 교차 확인 가능? | FEVER, IFCN |

> **D8(사용자 정의)**: 이 7개 어디에도 안 들어맞는 질문은 D8로 분류돼요. AI가 헷갈려서 임의로 끼워 맞추지 않게 하기 위함입니다.

## 3. 전체 흐름 한눈에 보기

```
[1] 내 체크리스트 만들기  ─────────────────►  질문마다 1~5점 루브릭 정의
                                                    │
                                                    ▼
[2] AI가 질문을 7대 차원에 자동 매핑  ───────► "이 질문은 D2(정확성)이네"
                                                    │
                                                    ▼
[3] 교사가 등록한 미디어를 내가 평가
        + 같은 미디어를 교사도 평가
                  │
                  ▼ (학생 vs 교사 점수 비교 = "격차 gap")
[4] 베이지안 가중치 갱신
        : 격차가 큰 차원의 가중치를 살짝 조정
                  │
                  ▼ 여러 번 반복 → 내 가중치 W가 안정됨
                  │
                  ▼
[5] 새 미디어를 Gemini가 7개 차원으로 1~5점 평가
                  │
                  ▼
[6] 최종 점수 = (W × Gemini 점수)를 50점 환산
        + 95% 신뢰구간(±오차범위)
                  │
                  ▼
[7] 결과 화면: 그대로 "수용" or 직접 점수 고쳐 "정교화"
```

## 4. 7층 구조로 다시 풀면

```
Layer 7 │ 최종 점수(50점 만점) + 95% 신뢰구간          ← 결과 카드
Layer 6 │ 베이지안 가중치 갱신 (μ 평균, σ 불확실성)     ← 학습이 일어나는 곳
Layer 5 │ 교사와의 격차 분석 → 메타인지 피드백 카드
Layer 4 │ 학생 가중치를 점수에 적용
Layer 3 │ Gemini가 7개 차원에 1~5점 부여 (단일 호출)
Layer 2 │ 체크리스트 항목 → 7대 차원 자동 매핑(캐싱)
Layer 1 │ 학생이 직접 만든 커스텀 체크리스트
```

> 위로 갈수록 "결과", 아래로 갈수록 "학생 본인의 입력"에 가깝습니다. 이 구조가 우리 모델의 이름 **HPFM (Hybrid Progressive Fact-Check Model)** 의 의미예요.
> - **Hybrid** = 학생 체크리스트 + 표준 7차원 + AI 평가 결합
> - **Progressive** = 평가를 반복할수록 점진적으로 내 기준이 학습됨

---

## 5. 핵심 개념 4가지

### (1) 가중치 W = { D1: {μ, σ}, …, D7: {μ, σ} }

각 차원마다 **두 개의 숫자**를 들고 다닙니다.

- **μ (뮤, mu)** = "이 차원을 얼마나 중요하게 볼 것인가"의 평균값. 7개의 μ를 다 더하면 1.
- **σ (시그마, sigma)** = "이 가중치에 대해 얼마나 자신 있는가"의 불확실성. 클수록 불확실, 작을수록 확신.

처음에는 모두에게 똑같이 1/7씩 나눠줍니다 (균등 가중치, σ=0.15).

### (2) 격차 (gap)

같은 미디어에 대해 **교사 점수 − 내 점수**.
- gap > 0 → 내가 너무 박하게 봤다
- gap < 0 → 내가 너무 후하게 봤다

### (3) 학습률 η(t)

> `η(t) = max(0.05, 0.2 × exp(−0.05 × t))`

- t = 누적 학습 데이터 수
- 처음(t=0)엔 0.2로 크게 움직이고, 데이터가 쌓일수록 천천히 작아집니다 (최소 0.05).
- 직관: **처음엔 빠르게 배우고, 나중엔 신중하게 미세 조정.**

### (4) 베이지안 갱신 식

> `μ_i_new = μ_i_old + η × gap_i × score_i / Σ(score_j²)`

말로 풀면:
1. 교사와 격차가 컸던 차원일수록 → 더 많이 움직인다
2. 그 미디어에서 강하게 두드러진 차원일수록 → 더 의미 있는 신호로 본다
3. 전체 점수 크기로 나눠서 → 모든 차원을 공평하게 비교한다

그리고 매 갱신마다 σ는 0.95배로 줄여요. **데이터가 쌓일수록 내 판단 기준이 안정화된다**는 의미입니다.

## 6. 학습 단계 정책

| 단계 | 누적 데이터 t | 무슨 일이 일어나나 |
|---|---|---|
| 🌱 Cold Start | t < 3 | 균등 가중치만 사용. 베이즈 갱신 비활성. (자료가 너무 적으면 섣불리 학습하지 않음) |
| 🌿 워밍업 | 3 ≤ t < 5 | 신중하게 갱신 시작 |
| 🌳 Bayesian 활성 | t ≥ 5 | σ 본격 감소, 가중치 정교화 가속 |

> 왜 이렇게 단계를 나눴을까? **표본이 너무 적을 때 학습하면 한 번의 우연한 격차에 휘둘려버리기** 때문이에요. 통계학에서 "작은 표본의 위험"이라고 부르는 문제를 막기 위함입니다.

## 7. 최종 점수와 신뢰구간

Gemini가 7개 차원에 1~5점을 매기면:

> `S = ( Σ μ_i × score_i ) × 10`        → 0~50점 만점

여기에 σ를 이용해 **얼마나 흔들릴 수 있는지**를 계산합니다.

> `Var(S) = 100 × Σ score_i² × σ_i²`
> `95% 신뢰구간 = S ± 1.96 × √Var(S)`

> **왜 ±인가?** 우리 모델은 단정적으로 "이 자료는 38점!"이라고 외치지 않아요. "38점인데, 95% 확률로 35.4 ~ 40.6 사이일 거야"처럼 **불확실성을 함께 보여주는 것**이 통계적으로 더 정직한 표현이기 때문입니다. 1.96은 정규분포에서 95% 구간을 만들 때 쓰는 표준값이에요.

## 8. "수용" vs "정교화"

결과를 본 학생은 두 가지 선택을 합니다.

| 선택 | 학습 데이터에 추가되는 값 | 가중치에 미치는 영향 |
|---|---|---|
| 🟢 **수용** | Gemini 점수 그대로 저장 | σ만 약간 ↓ (관찰이 누적된 효과) |
| 🟡 **정교화** | 학생이 직접 수정한 점수 | `bayesianUpdate` 발동, **학습률 η × 1.5 가속**, 학생-Gemini 격차를 신호로 사용 |

> "AI 결과에 동의하지 않는다"는 행위 자체가 **여러분의 비판적 판단**이고, 그것이 모델을 더 정교하게 다듬는 신호가 됩니다.

## 9. 메타인지 피드백 카드

여러 번의 격차 패턴을 분석해서 자동으로 만드는 진단 카드입니다 (`generateFeedbackCards`).

| 카드 | 트리거 조건 | 의미 |
|---|---|---|
| **Over** | 평균 gap < −0.5 | 교사보다 평균 0.5점 이상 **후하게** 평가하는 경향 |
| **Under** | 평균 gap > +0.5 | 교사보다 평균 0.5점 이상 **박하게** 평가하는 경향 |
| **Inconsistent** | gap 분산 > 1.0 | 평가 결과가 **흔들리는** 경향 (기준이 일관되지 않음) |

각 카드에는 차원·진단·개선 제안·관련 표준 프레임워크가 표시돼요.

> **메타인지(metacognition)** 란 "내가 어떻게 생각하고 있는지를 생각하는 능력"을 말합니다. 이 카드는 여러분 자신의 평가 습관을 거울처럼 비춰주는 도구예요.

## 10. 수렴도 (Convergence Score)

> `C = 1 − || W_student − W_teacher_implicit || / √7`

- 0 ~ 1 사이 값.
- 1에 가까울수록 **여러분의 가중치 분포가 교사가 (암묵적으로) 중요하게 본 차원 분포와 비슷해졌다**는 뜻.
- "성장 그래프"처럼 학생 대시보드에 표시됩니다.

---

## 11. 한 번의 평가, 어떻게 굴러갈까? (예시 시나리오)

**상황**: 학생이 "정확성·근거 제시"를 가장 중요하게 본다고 가정. 누적 평가 5회.

1. 학생이 미디어 A를 자기 체크리스트로 평가 → 차원별 점수 집계 → `D2=4, D4=3, ...`
2. 교사도 미디어 A를 평가 → `D2=2, D4=4, ...`
3. **gap = 교사 − 학생** → `D2=−2 (학생이 너무 후함), D4=+1 (학생이 약간 박함)`
4. 베이지안 갱신:
   - D2의 μ가 살짝 ↓ (정확성을 너무 후하게 봤으니 그 영향력 보정)
   - D4의 μ가 살짝 ↑ (근거 제시는 박했으니 그 영향력 강화)
5. 매 차원 σ ×0.95 → 전반적인 자신감 ↑
6. 정규화 (Σμ = 1)
7. 누적된 격차들을 보고 "Over D2" 같은 피드백 카드 자동 생성
8. 새 미디어 B를 Gemini로 평가 → 갱신된 W로 50점 환산 → `34.2 ± 4.1점`
9. 학생이 결과를 보고 "수용" 또는 "정교화" 선택 → 다시 7번으로 순환

## 12. 자주 묻는 질문

**Q. AI(Gemini)가 틀린 점수를 주면 어떻게 되나요?**
A. 그래서 "정교화" 버튼이 있습니다. 여러분이 동의하지 않으면 점수를 직접 수정하세요. 그 차이가 모델 학습 신호로 들어가서 다음 번엔 더 여러분에게 맞는 가중치가 됩니다.

**Q. 처음 한두 번은 점수가 이상해요.**
A. 정상입니다. 누적 데이터가 5개 미만이면 모델이 의도적으로 "균등 가중치"를 유지해요(Cold Start). 평가가 5번 이상 쌓이면 본격적으로 여러분 기준이 반영됩니다.

**Q. 7개 차원 중에 못 들어가는 질문이 있어요.**
A. AI가 자동으로 D8(사용자 정의)로 분류합니다. D8 항목은 가중치 학습엔 직접 영향을 안 주지만 여러분의 체크리스트엔 그대로 남아 있어요.

**Q. 신뢰구간이 너무 넓어요.**
A. 평가 횟수가 적어서 σ가 큰 상태입니다. 모델링을 반복할수록 σ가 줄어들고 신뢰구간도 좁아져요.

## 13. 더 알아보기 (출처·참고문헌)

이 알고리즘은 다음 4개의 정통 미디어 리터러시 프레임워크와 통계학의 베이지안 추론을 결합했습니다.

### 미디어 리터러시 프레임워크
- **CRAAP Test**: Sarah Blakeslee, *The CRAAP Test*, LOEX Quarterly 31(3), 2004. 정보 평가의 5요소(Currency, Relevance, Authority, Accuracy, Purpose)를 제시. California State University, Chico Meriam Library에서 정리.
- **SIFT (The Four Moves)**: Mike Caulfield, *SIFT (The Four Moves)*, 2019.
  Stop / Investigate the source / Find better coverage / Trace claims to original.
  공식 글: <https://hapgood.us/2019/06/19/sift-the-four-moves/>
- **IFCN Code of Principles**: International Fact-Checking Network (Poynter Institute).
  팩트체커 윤리·투명성 5원칙(Non-partisanship, Sources, Funding, Methodology, Corrections).
  공식: <https://ifcncodeofprinciples.poynter.org/>
- **FEVER (Fact Extraction and VERification)**: Thorne, Vlachos, Christodoulopoulos, Mittal, *FEVER: a large-scale dataset for Fact Extraction and VERification*, NAACL-HLT 2018. 자동 팩트체킹 벤치마크의 표준.
  공식: <https://fever.ai/>

### 통계·기계학습
- **Bayes' Theorem**: Thomas Bayes, *An Essay towards solving a Problem in the Doctrine of Chances*, Philosophical Transactions of the Royal Society of London, 1763 (사후 발표).
  새 증거가 들어왔을 때 신념(가중치)을 어떻게 업데이트할지에 대한 수학적 토대.
- **Online / Stochastic Gradient Update**: Robbins & Monro, *A Stochastic Approximation Method*, Annals of Mathematical Statistics, 1951.
  우리 모델의 학습률 스케줄(η decay)이 따르는 고전 결과.
- **95% Confidence Interval**: 일반적인 정규분포 가정 하의 신뢰구간 계산. 표준 통계학 교과서 어디에서나 다룸 (예: Wasserman, *All of Statistics*, Springer 2004, §6).

### AI / LLM
- **Google Gemini API**: 본 앱의 7차원 평가는 Gemini 2.5 Flash 모델을 Netlify Function 프록시로 호출.
  공식 문서: <https://ai.google.dev/gemini-api/docs>

### 코드에서 직접 확인하기
- 알고리즘 본체: `src/utils/hpfm.js`
- AI 호출(매핑/평가): `netlify/functions/gemini.js`
- 차원 매핑 캐싱: `src/utils/mappingCache.js`

## 14. 학습 활동 아이디어 (선생님과 함께)

1. **같은 기사, 다른 체크리스트**: 모둠별로 다른 7대 차원 강조 비율로 체크리스트를 만든 뒤, 같은 기사에 대해 어떤 결과 차이가 나오는지 비교.
2. **가중치 변화 추적**: 모델링을 5회, 10회, 20회 했을 때 각 차원의 μ 변화 그래프를 직접 그려보기.
3. **AI vs 인간**: Gemini의 7차원 평가를 그대로 수용한 결과와, 정교화로 수정한 결과를 비교해 "AI를 어디까지 신뢰할 것인가" 토론.
4. **메타인지 일지**: 피드백 카드(Over/Under/Inconsistent)를 받으면, 그 원인을 자기 언어로 정리하고 다음 평가에서 어떻게 다르게 해볼지 미리 적기.
